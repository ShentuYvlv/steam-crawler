## Oxylabs IP 轮换接入计划

### Summary
为现有 Steam 抓取和开发者回复发送链路接入 Oxylabs Datacenter 代理，并统一实现“抓取随机轮换、发送短会话固定 IP”的策略。

已确认的决策：
- 抓取：每次请求随机代理 IP
- 发送：单次发送任务内固定一个代理 IP
- 配置来源：全部从环境变量读取
- 代理失败：允许自动回退直连

核心目标：
- CLI 抓取链和后台任务链都能走代理
- 开发者回复发送链单独接入代理，不遗漏
- 代理轮换策略可控，不把凭据散落到代码或 YAML
- 后续能扩展为更多代理模式，不需要重构调用链

### Implementation Changes
#### 1. 新增统一代理配置与代理构造层
在现有配置体系上新增一组只读环境变量，供 CLI 与 backend 共用：
- `OXYLABS_PROXY_ENABLED`
- `OXYLABS_PROXY_HOST`
- `OXYLABS_PROXY_PORT`
- `OXYLABS_PROXY_USERNAME`
- `OXYLABS_PROXY_PASSWORD`
- `OXYLABS_PROXY_SCHEME`
- `OXYLABS_PROXY_ROTATING_PORT`
- `OXYLABS_PROXY_SESSION_PORT_MIN`
- `OXYLABS_PROXY_SESSION_PORT_MAX`
- `OXYLABS_PROXY_DIRECT_FALLBACK`

新增一个统一的“代理解析/代理 URL 生成器”模块，职责只做：
- 从 env 读取并校验代理配置
- 生成“随机轮换代理 URL”
- 生成“会话固定代理 URL”
- 根据策略返回 `httpx` 可直接使用的 proxy 配置
- 为发送任务生成一次性的 session port

约束：
- 抓取随机轮换固定用 `8000`
- 发送固定会话用 `8001..63000` 随机端口
- 不把代理账号密码写入 `config.yaml`

#### 2. 抓取链统一接入代理
在统一 HTTP 抽象层接入代理，而不是在每个 scraper 单独拼接：
- `AsyncHttpClient` 增加代理策略参数，至少支持：
  - `none`
  - `rotate_per_request`
  - `sticky_session`
- 默认抓取链使用 `rotate_per_request`
- 每次请求前动态获取代理配置，确保走 `8000` 随机 IP
- 保留现有重试、限流、超时逻辑，代理只是网络出口层

覆盖范围：
- 评论抓取
- 其他走 `AsyncHttpClient` 的 Steam 请求
- 探针/可用性检测是否走代理要和抓取一致，避免“探针与真实请求出口不同”导致误判

回退策略：
- 当代理请求失败且 `OXYLABS_PROXY_DIRECT_FALLBACK=true` 时，自动按现有重试策略尝试直连
- 日志里必须明确区分“代理请求失败后回退成功”与“全程直连成功”

#### 3. 开发者回复发送链单独接入固定会话代理
开发者回复发送当前没有走统一 HTTP 客户端，需要单独改：
- 在 `DeveloperReplyClient` 建立 `httpx.AsyncClient` 时接入代理
- 单次发送任务在开始时生成一个固定 session port，并在该任务生命周期内复用
- 同一个发送任务内不要中途换 IP
- 新建 client 时使用该固定代理；任务结束后关闭 client

这样发送链行为会变成：
- 单条回复任务：固定一个代理 IP 发完
- 后台恢复 pending 发送任务时：恢复后重新建立一个新的固定会话即可，不要求跨进程保留同一 IP

#### 4. 日志、观测与安全边界
为代理接入补齐结构化日志，但不打印明文密码：
- 当前请求是否启用代理
- 当前策略是随机轮换还是固定会话
- 当前端口类型是 `8000` 还是固定 session port
- 是否发生了代理失败后直连回退
- 代理失败的异常类型与次数

不记录：
- 完整代理 URL
- 明文用户名/密码

建议在后台任务日志里新增简短字段，便于排查：
- `proxy_enabled`
- `proxy_mode`
- `proxy_fallback_used`

### Public Interfaces / Types
需要新增或调整的接口/类型：
- 配置层新增代理配置模型，供 CLI 与 backend 共用
- `AsyncHttpClient` 构造参数新增代理策略/会话信息
- `DeveloperReplyClient` 构造参数新增代理配置或代理 URL
- 后台 env 说明文档和 `.env.example` 需要新增上述变量
- 若 backend 现有 settings 与 `src/config.py` 分离，需要在两边保持同一变量名与语义

默认策略定义：
- 抓取默认：`rotate_per_request`
- 发送默认：`sticky_session`
- fallback 默认：按你当前选择，允许直连回退

### Test Plan
必须补的测试：
- 抓取客户端在代理开启时，会为普通抓取请求使用 `8000`
- 发送客户端在代理开启时，会为单次发送任务选取固定 session port，并在任务内复用
- 代理关闭时，现有抓取和发送逻辑保持不变
- 代理失败且允许回退时，会继续尝试直连
- 代理失败且直连也失败时，错误会按现有链路抛出
- 探针请求与真实抓取请求使用同一代理策略
- 后台恢复 pending reply send 时，仍会重新建立代理并继续发送
- 日志中不包含明文代理密码

建议做的验证场景：
- 用 `https://ip.oxylabs.io/location` 做独立健康检查，确认：
  - 抓取模式每次请求 IP 可变化
  - 发送模式同一 client 内 IP 保持一致
- 本地 CLI 抓评论
- 后台手动同步评论任务
- 后台单条回复发送任务
- 代理故障时的回退路径

### Assumptions
- 代理接入目标仅限 Steam 抓取与 Steam 开发者回复发送，不扩展到 Aliyun LLM 调用
- CLI 与 backend 都使用同一组环境变量，不再把代理凭据放进 `config.yaml`
- 发送链采用“单次发送任务固定 IP”，即使你整体要求“乱换代理”，这里仍按你已选择的更稳方案处理
- 允许直连回退是最终行为，意味着在代理异常时源服务器 IP 可能暴露；实现中不会再额外加保护开关
